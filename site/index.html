three verbs

deploy — apps

Any repo with a Dockerfile, zero config. Services stay up; functions scale to zero and wake on request.

slab deploy owner/repo
deployed repo -> http://repo.localhost:8080

run — jobs

Run to completion: exit code back, logs kept, timeout guardrail. The sandbox for agent workflows.

slab run . -- npm test
slab run --node any repo -- make ci

up — systems

Apps wired on a private network. public = false members get no host port at all.

slab up ./system.toml
scoreboard -> @ garage (via trunk)

what's in the box

◉zero-manifest deploysname from the repo, port from the Dockerfile's EXPOSE. Paste a GitHub URL into the dashboard's empty bay.
◐functionsscale-to-zero with wake-on-request through the ingress. Idle apps cost nothing.
⧉systems + wiresprivate networks, env wiring, patch cables in the UI — wiring redeploys the caller automatically.
◫trunkssystems span machines: a per-system trunk carries http://member:port across nodes unchanged, private members included.
☀the solar systemzoom out and every node is a band — live status, dead nodes shown honestly. Any dashboard is a fleet console.
⚖cluster schedulingslab --node garage deploy… targets any peer; run --node any lands jobs on the idlest node.
▤skinnable dashboardhi-fi stereo rack by default, flat hyperscaler console built-in, or drop one CSS file in ~/.slab/skins.
⌁tunnels, postgres, secretsslab expose = free public HTTPS. postgres = true = DATABASE_URL appears. Secrets stay out of manifests.
